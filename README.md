# Market Basket Analysis for Retail Project

## Project Overview
This project focuses on identifying purchasing patterns and product associations using **Market Basket Analysis**. By analyzing transaction data, the goal is to discover which products are frequently bought together. This information helps businesses formulate cross-selling strategies, optimize store layouts, and improve targeted marketing campaigns.

## Dataset
The project utilizes transactional retail data.
- **Data Source**: The raw data is originally provided as `Assignment-1_Data.csv`.
- **Main Columns**:
  - **BillNo**: Unique identifier for each transaction.
  - **Itemname**: The name of the purchased item.
  - **Quantity**: The number of items purchased in the transaction.
  - **Date**: The date and time of the transaction.
  - **Price**: The price per item.
  - **CustomerID**: The unique identifier of the customer.
  - **Country**: The country where the transaction occurred.

## Project Structure
The repository contains the following main files and directories:
- `clean_data.ipynb`: A Jupyter Notebook containing the data wrangling pipeline. It cleans the raw data, handles missing values, removes duplicates, and standardizes formats before saving it as `Assignment-1_after_clean.csv`.
- `Assignment-1_after_clean.csv`: The cleaned dataset ready to be loaded into the database.
- `upload/`: A directory containing Python scripts (`import_customer.py`, `upload_country.py`, `upload_invoice.py`, `upload_invoice_detail.py`, `upload_product.py`) used to upload the cleaned data into a MySQL local database.
- `result.ipynb`: A Jupyter Notebook that connects to the MySQL local database, retrieves transaction data, and performs the Market Basket Analysis using the Apriori algorithm.
- `report.pbix`: A Power BI dashboard file that provides interactive visualizations of the dataset and the analysis insights.
- `item_names.txt`: A text file containing specific item names.

## Data Processing
The raw data is processed in `clean_data.ipynb`. The pipeline involves:
1. **Formatting**: Commas in the `Price` column are replaced with dots to correctly cast to float type. The `Date` is formatted to a standard `datetime` object.
2. **Missing Values**: Missing values in `CustomerID` are imputed by assigning new pseudo-customer IDs grouped by `BillNo`.
3. **Cleaning**: Rows with missing `Itemname` or negative `Price` values are removed. Duplicates are also identified and dropped.
4. **Calculated Field**: A `Revenue` column is generated by multiplying `Quantity` and `Price`.

## Methodology
The core algorithm used in this project is the **Apriori Algorithm**, which is implemented to find frequent itemsets and generate Association Rules.

### Key Concepts in Market Basket Analysis:
- **Apriori Algorithm**: An algorithm used to mine frequent itemsets and derive boolean association rules. It works iteratively, first finding individual items with minimum support, and then extending them to larger itemsets.
- **Support**: Measures how frequently an itemset appears in the dataset. (e.g., if Support is 0.02, the itemset appears in 2% of total transactions).
- **Confidence**: Measures the likelihood that an item B is purchased when item A is purchased. (e.g., A confidence of 0.8 means 80% of customers who bought item A also bought item B).
- **Lift**: Measures the strength of the association between item A and item B over the random occurrence. A Lift > 1 indicates that the items are highly correlated and frequently bought together.

In `result.ipynb`, `min_support` is configured to 0.02, and `min_threshold` for confidence is configured to 0.3.

## Dashboard / Visualization
The project includes a detailed Power BI dashboard (`report.pbix`). Providing clear visual data summaries such as total revenue, top-selling items, customer segmentation by country, and the strongest product associations over time. The interactive nature of Power BI makes reviewing revenue charts and cross-sell opportunities extremely accessible.

## How to Run the Project
1. **Set Up Database**: Ensure a local MySQL instance is running on port `3307`. Create a new database named `retail_project`.
2. **Clean Data**: Open the `clean_data.ipynb` notebook to review the data cleaning process.
3. **Upload Data**: Execute the Python scripts in the `upload` directory to populate the MySQL tables with the cleaned dataset.
4. **Run Analysis**: Open `result.ipynb`. Update the MySQL user, password, and port in the notebook's connection string if necessary. Run the notebook to generate the frequent itemsets and export association rules to CSV.
5. **View Dashboard**: Open `report.pbix` in Power BI Desktop to interact with the visualizations based on the generated data.

## Key Insights
- **Strong Product Associations**: Products like the *GREEN REGENCY TEACUP AND SAUCER* and *ROSES REGENCY TEACUP AND SAUCER* have high lift values (e.g., ~18), strongly indicating they are purchased simultaneously.
- **Theming the Basket**: Items like *GARDENERS KNEELING PAD KEEP CALM* and *GARDENERS KNEELING PAD CUP OF TEA* show strong relationships, indicating buyers have aesthetic or hobbyist preferences that could be leveraged for targeted bundles.
